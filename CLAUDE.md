## Layout

- `widget-src/` - editable widget sources (`widget.html`, `widget.css`, `widget.ts`, `globals.d.ts`). Bundled by Vite into the built skill's `assets/widget-bundled.html`. Not shipped directly.
- `skill-src/` - skill source content. Holds the `SKILL.md` template (with a `{{SCHEMA}}` placeholder), the JSON schema at `skill-src/assets/schema.json`, and the runtime scripts at `skill-src/scripts/` (`render.py` plus vendored chevron under `_vendor/`).
- `skill/linear-editing/` - the built skill directory, fully generated by the build pipeline from `widget-src/` and `skill-src/`. Contains `SKILL.md` (with the schema injected), `assets/widget-bundled.html`, `assets/schema.json`, `scripts/render.py`, and `scripts/_vendor/`. Committed to the repo so it appears in the zip and in GitHub browsing. The directory name is derived from `package.json` (strip the `claude-skill-` prefix).
- `tests/` - testing docs and eval scaffolding (manual checklists at the top, programmatic suite under `tests/evals/`)
- `vite.config.ts` - Vite + vite-plugin-singlefile config. Reads `package.json` to derive the skill name, bundles `widget-src/*` into `skill/<skill-name>/assets/widget-bundled.html`, and via the `assembleSkill` plugin assembles the rest: injects the schema into `SKILL.md`, copies `schema.json`, copies `scripts/`, and chmods `render.py` executable. Uses terser for JS, lightningcss for CSS. Run with `pnpm build`.
- `scripts/build-zip.sh` - runs `pnpm build`, then zips the `skill/<skill-name>/` folder from within `skill/` so the archive root matches the skill name.
- `package.json` / `tsconfig.json` / `pnpm-lock.yaml` - Node toolchain (requires Node 20+ and pnpm 10+). Widget JS is authored as strict TypeScript and stripped to JS at build time.

At skill runtime, Claude reads `SKILL.md` (which embeds the schema), constructs a JSON payload matching that schema, calls `python3 ${CLAUDE_SKILL_DIR}/scripts/render.py '<json>'`, and pipes stdout to `visualize:show_widget`. `render.py` uses vendored chevron (MIT-licensed Mustache implementation) for deterministic substitution.

## Before changing the skill, read the testing docs

[`tests/EVALS.md`](tests/EVALS.md) is the entry point. It explains the test surfaces (Vitest unit/integration, manual trigger walkthrough, programmatic eval suite, description tuning) and which to run based on what you're changing. **Do not skip it.** Changes to `widget-src/` should be backed by Surface 1 (Vitest, runs on every PR). Changes to `SKILL.md` or parsing rules should be backed by Surface 2 or Surface 3.

## Release pipeline

Release Please opens a release PR whenever a `feat:` or `fix:` commit lands on `main`. Merging the PR cuts a tag and uploads `linear-editing.zip` to the GitHub release. Conventional commit prefixes (`feat:`, `fix:`, `docs:`, `chore:`) determine the bump.

## Widget rendering caveat

The widget renders in claude.ai (web) and Claude Desktop. Claude Code, `claude -p` (headless CLI), and the Anthropic API cannot invoke `visualize:show_widget`, which the skill depends on. Any evaluation harness that depends on actual rendering needs to account for this - see [`tests/EVALS.md`](tests/EVALS.md) for details.
